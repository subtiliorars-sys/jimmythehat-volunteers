# Junior Game Developer (TypeScript)

**Role Description**

---

## Overview

Junior Game Developers work on implementing game features, fixing bugs, and contributing to the codebase of our racing game project. This is a learning-focused role with mentorship from experienced developers.

## Responsibilities

### Core Tasks
- Implement game features based on design specifications
- Fix bugs and issues reported by testers
- Write clean, documented code
- Participate in code reviews
- Test your changes thoroughly

### Learning Opportunities
- Learn TypeScript best practices
- Understand game development patterns
- Practice collaborative development
- Build a portfolio of real game code

## Requirements

### Technical Skills
- Basic programming knowledge (JavaScript or similar)
- Understanding of variables, functions, loops, and conditionals
- Willingness to learn TypeScript
- Familiarity with Git and GitHub (or willingness to learn)

### Soft Skills
- Good communication skills
- Ability to ask questions when unsure
- Willingness to learn from feedback
- Reliable and consistent

### Time Commitment
- 5-10 hours per week
- Minimum 2-month commitment preferred
- Flexible scheduling

## What You'll Work On

### Example Tasks
- Add a new power-up to the game
- Fix collision detection bugs
- Implement UI elements
- Optimize game performance
- Add sound effects integration
- Create player progression systems

### First Project
Your first task will be small and well-defined, such as:
- Fixing a simple bug
- Adding a small feature
- Improving existing code

## Tools and Technologies

### Development Environment
- **Language:** TypeScript
- **Version Control:** Git/GitHub
- **Code Editor:** VS Code (recommended)
- **Runtime:** Node.js

### Game Framework
- The project uses a custom TypeScript game engine
- You'll learn the engine's API and patterns
- Documentation and examples provided

## Mentorship

### What You'll Get
- Code review on all your contributions
- Regular check-ins with experienced developers
- Guidance on best practices
- Help when you're stuck
- Career advice if requested

### Mentorship Format
- Weekly check-in meetings (optional)
- Code review via GitHub pull requests
- Real-time help via Discord/email
- Learning resources and tutorials

## Growth Path

### Skill Development
As you gain experience, you can take on:
- More complex features
- Architecture decisions
- Mentoring newer volunteers
- Leading small features

### Portfolio Building
All your contributions will be:
- In a public GitHub repository
- Attributed to you in commit history
- Included in release notes
- Usable as portfolio pieces

## Success Metrics

### What Success Looks Like
- Completing assigned tasks on time
- Writing clean, readable code
- Learning from feedback
- Asking questions when needed
- Contributing to team discussions

### Evaluation
- Regular feedback from mentors
- Code quality assessments
- Task completion rate
- Communication reliability

## FAQ

**Do I need to know TypeScript?**
No, but you should know basic programming. We'll help you learn TypeScript.

**What if I get stuck?**
Ask for help! We have mentors available via Discord and email.

**Can I choose what I work on?**
To some extent. We'll assign tasks based on project needs, but we'll consider your interests.

**How long is the commitment?**
We prefer at least 2 months, but it's flexible. The most important thing is reliable communication.

**Can this lead to paid work?**
While we can't guarantee paid work, the experience and portfolio pieces can help you get paid positions elsewhere.
