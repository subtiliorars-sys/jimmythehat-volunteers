# Game Designer

**Role Description**

---

## Overview

Game Designers work on the mechanics, systems, and player experience of our racing game. This is a learning-focused role with mentorship from experienced designers.

## Responsibilities

### Core Tasks
- Design game mechanics and systems
- Create level layouts and progression
- Write game design documents
- Playtest and balance gameplay
- Collaborate with developers and artists

### Learning Opportunities
- Learn game design principles
- Understand player psychology
- Practice systems thinking
- Build a portfolio of design work

## Requirements

### Design Skills
- Understanding of game mechanics and systems
- Ability to think from the player's perspective
- Good communication skills
- Creativity and problem-solving

### Technical Skills
- Basic understanding of game development process
- Familiarity with games in the racing genre
- Ability to write clear documentation
- Familiarity with version control (Git) helpful but not required

### Soft Skills
- Good communication and collaboration
- Ability to give and receive feedback
- Analytical thinking
- Reliable and consistent

### Time Commitment
- 5-10 hours per week
- Minimum 2-month commitment preferred
- Flexible scheduling

## What You'll Work On

### Example Tasks
- Design power-up mechanics
- Create level layouts
- Balance vehicle stats
- Design progression systems
- Write game design documents
- Conduct playtesting sessions

### First Project
Your first task will be small and well-defined, such as:
- Documenting an existing mechanic
- Designing a simple power-up
- Creating a level layout

## Tools and Technologies

### Documentation
- **Writing:** Google Docs, Notion, or Markdown
- **Diagrams:** Draw.io, Miro, or similar
- **Spreadsheets:** Google Sheets or Excel (for balance)

### Collaboration
- **Communication:** GitHub Discussions, Discord
- **Version Control:** GitHub for documentation
- **Project Management:** GitHub Issues and Projects

## Design Process

### 1. Research and Analysis
- Study existing racing games
- Analyze what works and what doesn't
- Understand player expectations

### 2. Concept Development
- Brainstorm ideas
- Create initial concepts
- Get feedback from mentors

### 3. Documentation
- Write clear design documents
- Include diagrams and examples
- Specify requirements clearly

### 4. Implementation Support
- Work with developers to implement designs
- Adjust based on technical constraints
- Test and iterate

### 5. Playtesting and Balance
- Conduct playtesting sessions
- Gather feedback
- Adjust balance and mechanics

## Design Principles

### Project Design Philosophy
The project follows these design principles:
- **Fun First:** Mechanics should be enjoyable above all
- **Accessibility:** Easy to learn, hard to master
- **Feedback:** Clear player feedback for all actions
- **Progression:** Meaningful advancement systems

### Player Experience
All designs should consider:
- What the player feels
- What the player understands
- What the player can do
- What the player wants to do next

## Mentorship

### What You'll Get
- Design guidance and feedback
- Regular check-ins with experienced designers
- Help with design documentation
- Guidance on playtesting methodology
- Portfolio advice if requested

### Mentorship Format
- Weekly check-in meetings (optional)
- Design review via GitHub Discussions
- Real-time help via Discord/email
- Learning resources and articles

## Growth Path

### Skill Development
As you gain experience, you can take on:
- More complex systems
- Lead design for features
- Mentoring newer designers
- Contributing to overall game direction

### Portfolio Building
All your contributions will be:
- In a public GitHub repository
- Attributed to you in documentation
- Included in game credits
- Usable as portfolio pieces

## Success Metrics

### What Success Looks Like
- Creating clear, implementable design documents
- Designing fun and balanced mechanics
- Incorporating feedback effectively
- Collaborating well with developers and artists
- Contributing to cohesive game experience

### Evaluation
- Regular feedback from mentors
- Design quality assessments
- Implementation success rate
- Player feedback on designs

## FAQ

**Do I need to be a game design expert?**
No! We welcome designers of all experience levels. The most important thing is passion and willingness to learn.

**Do I need to know how to code?**
No, but you should understand the development process. You'll work closely with developers.

**Can I choose what I work on?**
To some extent. We'll assign tasks based on project needs, but we'll consider your interests.

**How do I share my designs?**
You'll write design documents in the project repository. We'll help you with the format.

**Can this lead to paid work?**
While we can't guarantee paid work, the experience and portfolio pieces can help you get paid positions elsewhere.
