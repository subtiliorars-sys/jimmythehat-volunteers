# Getting Started - Volunteer Onboarding

**Welcome to JimmyTheHat Games!**

This guide will help you get set up and start contributing.

---

## Before You Begin

### Prerequisites

**For Developers:**
- Basic programming knowledge (JavaScript/TypeScript helpful but not required)
- Git and GitHub account
- Code editor (VS Code recommended)
- Node.js and npm (for TypeScript projects)

**For Artists:**
- Digital art software (Photoshop, GIMP, Krita, etc.)
- Basic understanding of file formats (PNG, SVG, etc.)
- GitHub account for asset sharing

**For Designers:**
- Note-taking software (Google Docs, Notion, etc.)
- GitHub account for documentation
- Basic understanding of game mechanics

---

## Step 1: Acceptance and Setup

### You've Been Accepted! Now What?

1. **Check Your Email**
   - You'll receive an invitation to the project's GitHub organization
   - Accept the invitation to gain repository access

2. **Join Communication Channels**
   - GitHub Discussions (primary communication)
   - Email for important updates
   - Discord (if applicable to your role)

3. **Set Up GitHub**
   - Make sure your GitHub profile is complete
   - Add a profile picture
   - Enable two-factor authentication (recommended)

---

## Step 2: First Tasks

### Your First Assignment

**Week 1 Goals:**
- Set up your development environment
- Review project documentation
- Complete a small, well-defined task
- Introduce yourself to the team

**Specific First Task:**
You'll be assigned a small task based on your role:
- **Developers:** Fix a simple bug or add a small feature
- **Artists:** Create a simple asset or modify an existing one
- **Designers:** Review and document a game mechanic

---

## Step 3: Development Workflow

### For Developers

**1. Clone the Repository**
```bash
git clone https://github.com/subtiliorars-sys/DrivingMeNuts.git
cd DrivingMeNuts
```

**2. Install Dependencies**
```bash
npm install
```

**3. Run the Project**
```bash
npm start
```

**4. Create a Branch**
```bash
git checkout -b feature/your-task-name
```

**5. Make Changes**
- Edit files
- Test your changes
- Commit your work

**6. Submit a Pull Request**
- Push your branch
- Create a pull request on GitHub
- Request review from a mentor

### For Artists

**1. Access Assets**
- Check the project's assets folder
- Review existing style guides
- Understand the art direction

**2. Create Your Work**
- Follow the style guide
- Use appropriate file formats
- Name files clearly

**3. Share Your Work**
- Upload to the project repository
- Create a pull request or issue
- Request feedback

### For Designers

**1. Review Documentation**
- Read existing game design documents
- Understand the current game mechanics
- Review player feedback

**2. Create Your Design**
- Write clear, specific design documents
- Include examples and diagrams
- Consider implementation feasibility

**3. Share Your Design**
- Post in GitHub Discussions
- Create an issue for feedback
- Iterate based on feedback

---

## Step 4: Communication

### How to Communicate

**Daily Updates:**
- Post a brief update in GitHub Discussions
- Mention what you worked on
- Note any blockers or questions

**Questions:**
- Post questions in GitHub Discussions
- Tag mentors for specific questions
- Search existing discussions first

**Feedback:**
- Give constructive feedback on others' work
- Be open to feedback on your work
- Focus on improving the project

### Response Time Expectations

- **You:** Respond to messages within 48 hours
- **Mentors:** Respond within 24-48 hours
- **Urgent Issues:** Contact via email for faster response

## Step 7: Getting Help

### When You're Stuck

1. **Search First**
   - Check existing GitHub Issues and Discussions
   - Look at project documentation
   - Search online for similar problems

2. **Ask a Question**
   - Post in GitHub Discussions
   - Be specific about your problem
   - Share what you've tried

3. **Contact a Mentor**
   - Tag a mentor in your discussion
   - Send an email for urgent issues

### Common Issues

**Can't Build the Project:**
- Check that you have the correct Node.js version
- Delete node_modules and run `npm install` again
- Check for error messages in the console

**Don't Understand a Task:**
- Ask for clarification in GitHub Discussions
- Request a meeting with a mentor
- Look at similar completed tasks

**Running Out of Time:**
- Communicate early if you can't complete a task
- Ask for help prioritizing
- It's okay to say you're busy

## Step 8: Tracking Your Hours

### For School Credit

If you need documentation of volunteer hours:

1. **Log Your Work**

2. **Request Documentation**

   - We'll provide a summary of your contributions
