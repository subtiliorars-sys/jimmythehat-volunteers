# Contributing to JimmyTheHat Games

**Thank you for your interest in contributing!**

---

## Getting Started

### First Time Contributors

1. **Read the README**
   - Start with the [main README](README.md) for an overview
   - Review the [project overview](docs/projects/drivingmenuts.md)
   - Check out [role descriptions](docs/roles/) to find what fits you

2. **Apply to Volunteer**
   - [Create a volunteer application](https://github.com/subtiliorars-sys/jimmythehat-volunteers/issues/new/choose)
   - We'll review within 48 hours
   - Once accepted, you'll get repository access

3. **Complete Onboarding**
   - Follow the [getting started guide](docs/onboarding/getting-started.md)
   - Set up your development environment
   - Join our communication channels

---

## How to Contribute

### For Developers

1. **Find an Issue**
   - Look for issues labeled "good first issue" or "help wanted"
   - Comment on the issue to claim it
   - Wait for assignment

2. **Create a Branch**
   ```bash
   git checkout -b feature/descriptive-name
   ```

3. **Make Your Changes**
   - Write clean, documented code
   - Follow the project's code style
   - Test your changes thoroughly

4. **Submit a Pull Request**
   - Push your branch
   - Create a pull request
   - Describe your changes clearly
   - Request review from a mentor

### For Artists

1. **Find a Task**
   - Look for issues labeled "art needed"
   - Comment on the issue to claim it
   - Wait for assignment

2. **Create Your Art**
   - Follow the style guide
   - Use appropriate file formats
   - Optimize for performance

3. **Share Your Work**
   - Upload assets to the repository
   - Create a pull request or issue
   - Request feedback from mentors

### For Designers

1. **Find a Task**
   - Look for issues labeled "design needed"
   - Comment on the issue to claim it
   - Wait for assignment

2. **Create Your Design**
   - Write clear documentation
   - Include examples and diagrams
   - Consider implementation feasibility

3. **Share Your Design**
   - Post in GitHub Discussions
   - Create an issue for feedback
   - Iterate based on feedback

## Communication

### Where to Communicate

- **GitHub Discussions:** For questions, ideas, and general discussion
- **GitHub Issues:** For bugs, tasks, and specific contributions
- **Email:** For private matters or urgent issues

### Response Time Expectations

- **You:** Respond within 48 hours
- **Mentors:** Respond within 24-48 hours
- **Urgent:** Email for faster response

---

## Getting Help

### When You're Stuck

1. **Search First**
   - Check existing issues and discussions
   - Look at project documentation
   - Search online for similar problems

2. **Ask a Question**
   - Post in GitHub Discussions
   - Be specific about your problem
   - Share what you've tried

3. **Contact a Mentor**
   - Tag a mentor in your discussion
   - Send an email for urgent issues
